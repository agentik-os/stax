# U-026: the terminal panel · the chat panel (modern surfaces in the grammar)

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/PlatformApp.tsx (term, pfchat) + styles.css

## DETECT
- CLI/console and AI-chat features live as modals, embedded iframes or a
  detached chrome widget instead of PANELS in the thread.

## APPLY
1. TERMINAL PANEL: mono scrollback as the body (accent ❯ command echoes,
   destructive-red errors, auto-scroll), the PROMPT as the FOOT (the action
   zone: a borderless mono input + a quiet Clear). ArrowUp/Down walk the
   command history; Escape blurs the prompt, never the panel. The verbs bind
   to the copilot bridge (status · open <q> · close · pin · undo · redo ·
   actions · run <id> · tour · theme) so the terminal is a REAL third control
   surface — `open x` navigates, and the terminal itself travels with its
   space (pin it to keep it: the thread law applies to every surface).
2. CHAT PANEL: a conversation as CONTENT — messages in the body (mono speaker
   labels, YOU in accent, hairline separators, a typing beat before replies),
   the composer in the foot with one primary Send. Slash commands execute
   through the bridge. The global agent drawer remains chrome; chat SESSIONS
   are rows/panels like any other content.

## VERIFY
- `status` prints the live workspace (space, thread, rail); `open <title>`
  really opens the panel (and moves the thread); ArrowUp restores the last
  command. The chat shows the typing beat then replies; /actions lists the
  focused panel's registry actions.
