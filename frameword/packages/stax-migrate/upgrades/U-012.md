# U-012: separator dedup · neutral focus ring · resizable pinned refs

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/styles.css (separator dedup block,
button:focus-visible) + App.tsx (ref width)

## WHY
Three finishing rules learned in production: stacked hairlines double up at block
boundaries; Safari fires :focus-visible on CLICK (accent rings appear on every tap);
and rail references were hard-capped at S, ignoring the user's size preference.

## DETECT
- Two hairlines closer than 3px anywhere in a panel body (scan border edges).
- An accent outline appears around buttons/segments after a mouse click (Safari).
- The width control has no effect on a pinned reference.

## APPLY
1. ONE separator between any two stacked blocks: section containers absorb their first
   child's top hairline (drills, lists, definition rows, cards); every list drops the
   LAST row's bottom hairline (drills, entity lists, hub cards).
2. Focus ring: neutral color (foreground mix, never accent) and suppressed while
   hovered/active (button:hover:focus-visible { outline: none }) — keyboard keeps it.
3. References default to S but the per-panel size preference applies in pin mode too.

## VERIFY
- Geometric scan: zero hairline pairs ≤ 3px apart on three dense panels.
- Click a segment in Safari/WebKit: no ring; Tab to it: neutral ring.
- Pin a panel, set L via the gear: the rail panel widens (and remembers).
