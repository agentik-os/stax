# U-019: the action registry · contextual palette · layouts & recents

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (panelActions, Palette, noteRecent)

## DETECT
- Foot buttons are hand-written per panel type; the palette cannot surface a
  panel's actions; actions have no single declaration.
- No way to save/restore a whole workspace arrangement; no recent-threads list.

## APPLY
1. ACTION REGISTRY: one `panelActions(panel, node, id, ws): PanelAction[]`
   ({id, label, icon, kind: primary|secondary|destructive, run}) per panel type.
   The FOOT renders from it (kind maps to the foot hierarchy); the ⌘K palette
   prepends a CONTEXTUAL section ("<Panel title>: <action>") for the focused
   panel; nothing else declares actions.
2. RECENT THREADS: every settled thread records {spaceId, [{t,k}] path, leaf
   title} (device-local, cap 8, dedup by signature); the palette lists
   "Recent: <title>" entries reopening via openPath.
3. SAVED LAYOUTS: "Save current layout" snapshots the FULL WorkspaceState
   (auto-named leaf title + time, cap 6); "Layout: <name>" restores it through
   ws.restore() (validated, undoable); "Clear saved layouts" empties the store.

## VERIFY
- A foot's buttons and the palette's contextual entries come from the SAME
  registry call (grep: one panelActions definition, zero hand-written foot
  branches for registry types).
- Open A→B→C, then elsewhere; ⌘K shows "Recent:" for the old thread and
  reopens the full chain. Save a layout, mutate the workspace, restore: the
  exact panel set returns; mod+Z undoes the restore.
