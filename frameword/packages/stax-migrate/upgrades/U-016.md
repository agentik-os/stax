# U-016: panel search in the bar · quiet secondary buttons

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (searchable, .panel-search)
+ styles.css (.panel-search, .panel-foot hierarchy)

## DETECT
- List panels offer no way to search/filter their rows in place, or the search
  input lives in the FOOT (the action zone).
- Every foot button renders in the accent family: the secondary action is
  indistinguishable from the primary, and destructive looks like a CTA.

## APPLY
1. PANEL SEARCH AS CHROME: every list panel (4+ children, or an app list such as
   a tables home / notes / tasks) gets a ⌕ button in the panel BAR (before the
   pin) toggling a flat row under the bar: borderless input (autofocus, hairline
   bottom, a quiet × to close), Escape closes the ROW never the panel. The query
   live-filters the panel's lists (title + subtitle, case-insensitive); zero
   matches render a one-sentence empty state ("No matches: clear the search
   above."). Device-local state, never navigation state.
2. FOOT HIERARCHY: ONE primary per foot (accent filled + accent border).
   Secondary actions: transparent bg, 1px `--border`, ink-2 text; hover =
   secondary fill + foreground text. Destructive: `--destructive` TEXT on
   transparent chrome; hover = ~10% destructive fill. Kill any rule that forces
   all foot buttons into the accent family.

## VERIFY
- Open a list panel: ⌕ in the bar; typing narrows rows live (count proof);
  garbage query shows the empty state; Escape closes the row, the panel stays.
- Computed styles in a foot with 2+ actions: secondary borderColor == --border
  and color != accent; destructive color == --destructive with transparent
  background; exactly one accent-filled button.
