# U-018: a11y baseline (announce · arrows · focus return)

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (role=status, arrow nav)
+ DataApp.tsx RowSheet (focus land/return)

## DETECT
- Toasts are visual-only (no aria-live); screen readers never hear them.
- Drill lists are Tab-only (no arrow-key walking).
- Overlays (sheet, drawer) leave focus behind them on open and drop it on close.

## APPLY
1. The toast slot renders inside a `role="status"` `aria-live="polite"` region.
2. ArrowUp/ArrowDown move focus between the `.drill` rows of the list the
   focused row belongs to (never crossing lists; edges stay put).
3. Sheet and drawer: on mount, focus the first focusable inside; on unmount,
   RETURN focus to the element that opened them. Escape already closes the
   overlay, never the panel behind.
4. Every icon-only control (bar ⌕, ×, pin, chips) carries a title or aria-label.

## VERIFY
- Axe/manual: the status region exists once; triggering any action announces.
- Focus a drill, ArrowDown: focus moves to the next row (activeElement proof).
- Open the sheet: activeElement is inside it; Escape: the opener regains focus.
