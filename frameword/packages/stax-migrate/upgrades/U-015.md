# U-015: sidebar space KPIs · three-tier responsive shell

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (.sb-kpi, sbNarrow) + styles.css

## DETECT
- A space's important KPIs exist only inside its root panel: nothing keeps them in
  reach once the user drills deep or scrolls the stage.
- Between ~640 and ~900px the docked sidebar starves the stage (240px + an L 640
  panel never fit side by side), forcing permanent horizontal scroll.
- Stat labels clip raw at narrow widths; fixed drawers overflow small frames.

## APPLY
1. SIDEBAR SPACE KPIs: when the active space's ROOT node declares kpis, mirror them
   in the sidebar just ABOVE the pipeline/usage block: equal grid tracks
   (minmax(0,1fr)), mono tabular values under mono uppercase labels (label first,
   same order as .stats), vertical hairlines between tracks, and a FULL-WIDTH
   hairline below the block (same spans as the foot's top hairline) so the pipeline
   keeps its own separator. Single source of truth: the root's kpis, never a second
   dataset. No kpis → the block does not render.
2. THREE-TIER RESPONSIVE SHELL: >=900 docked sidebar; 640-900 the sidebar
   auto-closes and reopens as an OVERLAY with backdrop, EVERY nav click (head row
   included) closes it; <=640 PushHost + overlay sidebar with the dashboard
   switcher; <=760 hide the topbar nav. Fixed surfaces (drawer, sheet, palette)
   clamp to 88-92% of the frame width.
3. Stat labels (.stat .lab and the sidebar KPI labels): nowrap + ellipsis, never
   raw clipping.

## VERIFY
- Open a space whose root has KPIs: the sidebar shows them above the pipeline with
  the full-width hairline (rect proof: separator spans == foot hairline spans);
  a root without kpis renders no block.
- At 744px: sidebar auto-closed, stage takes the full width; the toggle opens an
  overlay + backdrop; clicking ANY sidebar row (head included) closes it.
- At 980px the sidebar is docked. At 375px no horizontal document overflow and
  stat labels ellipsise.
