# U-023: exit ghosts · far-left foot search · row virtualization

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (animateExit) + styles.css

## DETECT
- Closed panels vanish instantly: the one moment the grammar's object
  permanence breaks. The foot ⌕ floats right of the actions; long drill lists
  render every offscreen row.

## APPLY
1. EXIT GHOSTS: explicit close gestures (the bar ×, the compact back ‹,
   ctrl+X, Escape on the leaf, and the bridge's close()) clone the panel into a fixed ghost at its exact rect and animate it
   out (150ms, opacity + 0.98 scale, the system ease) via WAAPI; remove on
   finish AND cancel; `prefers-reduced-motion` skips entirely. Engine-driven
   removals stay instant. The clone carries a marker class and every panel
   query excludes it (`:not(.panel-exit)`).
2. FOOT SEARCH FAR LEFT: the ⌕ toggle is the foot's FIRST element; the active
   input starts from the same left edge, so both states share the anchor.
3. VIRTUALIZATION: drill rows get `content-visibility: auto` with a
   `contain-intrinsic-size` matching the row height, so 500-row lists stop
   rendering offscreen content.

## VERIFY
- Click ×: a .panel-exit ghost appears and is gone within ~200ms; with
  reduced-motion emulated, none appears. Space switches spawn no ghosts.
- The ⌕ TOGGLE and the active search row share the same left EDGE (element
  boxes, not glyph positions: the 28px toggle centers a 14px glyph).
- Computed contentVisibility on a drill row is "auto"; the alignment gate
  still passes (measurements force layout).
