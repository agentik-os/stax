# U-024: the view grammar — one dataset, four shapes · foot control deck

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/DataApp.tsx (View.type, dtb/dtc/dtl)
+ App.tsx (DtFootSeg) + styles.css

## DETECT
- The app renders its collections as separate pages per layout (a kanban page, a
  table page, a cards page) instead of VIEW TYPES over one dataset; filters and
  layout switchers float in headers; hero KPIs shout in display sizes.

## APPLY
1. VIEW TYPES: each saved view carries type table | board | cards | list over
   the SAME filtered/sorted rows. BOARD: the first select field's options are
   the columns (+ a trailing no-value column when occupied), drag a card to
   restage (writes the field), column = secondary bg radius 10, head = mono
   label + tabular count, empty = dashed "Drop here". CARDS: auto-fill grid of
   flat cards (title + up to 4 label/value lines), click = peek sheet. LIST:
   one-line rows in the drill grammar (title · select pill · number right).
2. FOOT CONTROL DECK: the view-type switcher lives in the FOOT as segments
   (.foot-seg: hairline container, mono uppercase labels, card-bg active) with
   the live row count as a right-aligned foot note; the same pattern hosts any
   global filter (fund switcher, archived toggle). Also available from the
   active view tab's menu ("View as").
3. QUIET HERO KPIs: stat values are 17px mono (the sidebar KPI mirror carries
   the numbers); display-size digits never sit in a panel hero.

## VERIFY
- Switch a view to Board: columns match the select options; dragging a card to
  another column updates the cell (store proof); Cards and List render the same
  filtered rows; the foot segments and the tab menu stay in sync.
- The foot shows the row count; the hero stat value computes to 17px.
