# U-004 — Dedicated space menus: the sidebar mirrors the OPEN space

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (sbSpace/sbView, .sb-space-head,
.sb-backarrow, .sb-collapse, .panel.root-collapsed) · styles.css

## WHY
When a space is open, repeating the dashboard's space list in the sidebar wastes the
menu. The sidebar becomes the OPEN space's own menu (the console "Back to project"
pattern): its children as numbered deep-links, with a back arrow to the dashboard list.

## DETECT
- Opening a space leaves the sidebar showing the same flat list of all spaces.

## APPLY
1. State: sbView "dash" | "space" — auto-switches to "space" when the active spaceId
   belongs to the current dashboard; the back arrow returns to "dash".
2. Head row (ONE row, geometrically stable): ‹ back arrow (26px icon, INSTANT custom
   tooltip = the dashboard name — the native title is too slow) + the space label as a
   plain sidebar row (sans 600 — NEVER serif in chrome) that focuses/reveals the root +
   a collapse toggle ALWAYS rendered, disabled (opacity .35) while the root is alone.
3. Items: the root's children, mono-numbered 01/02…, each a real deep-link; one-level
   expansion to grandchildren (chevron) when ≤ 14; active thread highlighted from the
   context path.
4. Root show/hide: the toggle collapses the ROOT panel to a ~46px spine on stage
   (vertical serif title, click/Enter to expand). UI-only state — never navigation
   state; resets on space switch.

## VERIFY
- Open a space → dedicated menu; back arrow → dashboard list; hover arrow → tooltip
  appears < 300ms. Screenshots.
- Head geometry identical (rects) with root alone vs after drilling.
- Collapse → spine ≈ 46px; click → expands; switch space → reset.
