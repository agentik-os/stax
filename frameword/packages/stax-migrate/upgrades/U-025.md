# U-025: click-through filters · one-row toolbar · cell walking · team tasks

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/DataApp.tsx (filter chips, walk) +
NotesApp.tsx (TasksRoot groups) + styles.css

## DETECT
- Filters are typed into text inputs; the table burns two toolbar rows; cell
  editing has no keyboard flow; tasks have no assignee/project dimension and
  only one grouping.

## APPLY
1. FILTERS ARE CLICKED, NOT TYPED: the Filter menu lists fields; picking one
   opens per-type value pickers (select/multiselect options as ✓ toggles with
   an "is any of" op — value joined by "|"; dates via the popover calendar +
   on/before/after segments; check as two segments; numbers/text keep a small
   input for the value only). Active filters live as toolbar CHIPS (mono field
   label + value summary; click the body to edit, × to drop).
2. ONE-ROW TOOLBAR: view tabs, filter chips, sort/group chips and tools share
   a single wrapping row — never two stacked toolbar bands.
3. CELL WALKING: click anywhere in a td to focus its editor; Enter commits and
   moves DOWN the same column (skipping group rows); Escape blurs the cell and
   never reaches the panel; chrome controls (peek, actions, checkboxes) leave
   the Tab order (tabIndex -1) so Tab walks data only.
4. TEAM TASKS: Task gains assignee/priority/project; the team's first member is
   the CONNECTED profile. One group control (status/person/priority/project)
   drives list sections AND kanban columns; a kanban drop WRITES the grouped
   field (reassign by drag). Click-through people/project filter chips +
   a "Mine" toggle scoped to the connected profile. Assignees render as
   18px initial avatars, projects as ghost mono tags.

## VERIFY
- Build "Plan is any of Pro, Scale" with FOUR clicks and zero typing; the chip
  reads the summary; × restores all rows.
- Toolbar rects: tabs and Filter button share the same top (±10px).
- Click a cell's padding: its input focuses; Enter focuses the same column one
  row down; Escape leaves the panel count unchanged.
- Group by Person: sections/columns are the team, dragging a card to another
  person rewrites task.assignee; "Mine" shows only the connected profile's.
