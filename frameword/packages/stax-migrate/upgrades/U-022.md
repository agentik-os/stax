# U-022: devtools time travel · shareable workspace links · "?" map

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (DevtoolsBody, encodeShare) +
packages/panels-react (history)

## DETECT
- No way to inspect the live workspace state, replay how it got there, or hand
  a colleague the EXACT arrangement; shortcuts are undiscoverable.

## APPLY
1. DEVTOOLS as a sys PANEL (the workspace inspecting itself): live state JSON
   (+ copy), the provider's history() (undo stack, oldest first: each entry =
   the state an intent replaced) as drill rows, Jump = ws.restore(entry)
   (validated, itself undoable), export actions in the flat grammar.
2. SHAREABLE LINKS: "Copy workspace link" gzips the FULL state into
   #ws=<base64url> (CompressionStream, no deps, no backend). Consume it at
   boot AND on hashchange (a link pasted into a live session must restore
   too); after restore the provider rewrites the hash to the thread encoding.
3. The "?" overlay lists the keyboard map (palette chrome, inert while
   typing); Escape closes the overlay BEFORE anything else.

## VERIFY
- Devtools history shows one row per intent; Jump changes the workspace to
  that state and mod+Z returns. A #ws link opens cold AND pasted live into an
  existing session; the hash never stays #ws after restore. "?" opens the map,
  Escape closes only it.
