# U-017: workspace undo · table multi-select & quick-search

TARGET: {{TARGET}}
REFERENCE: frameword/packages/panels-react/src/index.tsx (undo/redo stacks)
+ frameword/apps/crm-specimen/src/DataApp.tsx (selRows, dt-bulk)

## DETECT
- Closing or moving a panel is irreversible (no workspace history); ctrl+X makes
  accidental closes cheap but recovery impossible.
- Table rows cannot be selected in bulk; the row query lives in a per-table input
  instead of the panel's bar search.

## APPLY
1. UNDO/REDO: the provider records the PREVIOUS state on every intent (bounded
   stack, ~50; a no-op intent records nothing) and exposes undo()/redo().
   mod+Z / shift+mod+Z at the shell level, inert while typing (native undo) and
   YIELDED to a focused canvas that owns its own board history. Toast announces
   "Undone"/"Redone".
2. TABLE MULTI-SELECT: quiet checkboxes in the actions column (revealed on row
   hover or while a selection is live; header checkbox = select-all VISIBLE
   rows); selected rows tint 4% accent; a BULK BAR replaces the toolbar's left
   side (mono count · Duplicate · Delete as red text · clear ×). Batch store ops
   (removeRows, duplicateRows), never N single calls.
3. TABLE QUICK-SEARCH: the panel bar ⌕ feeds the table's row query (all text
   cells, case-insensitive) on top of the view's filters; the per-table input is
   removed.

## VERIFY
- Open a child, ctrl+X, mod+Z: the panel returns (count proof); shift+mod+Z
  re-closes. Typing in a cell, mod+Z never touches the workspace.
- Select 2 rows: bulk bar shows "2 selected"; Duplicate adds 2; Delete removes
  them; header checkbox selects exactly the visible (filtered) rows.
