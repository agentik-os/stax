# U-013: tables v3 · Notion row peek · the entity sheet

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/DataApp.tsx (RowSheet, view menu, openchip)
+ styles.css (.sheet*, .sh-*, .dt-openchip)

## DETECT
- Table views have no settings surface (no rename/duplicate/density/reset from the tab).
- No CSV export of the current view; New row opens a detail panel (layout jump).
- Rows have no hover peek affordance; record detail exists only as a full panel.
- No per-row activity (notes/tasks) structure.

## APPLY
1. VIEW SETTINGS: clicking the ACTIVE view tab opens its menu: Rename · Duplicate view ·
   Density Cozy/Compact (persisted PER VIEW) · Reset view · Delete view (>1 only).
2. ⋯ menu: Export CSV (visible fields, filtered+sorted rows, real download) · Reset.
   Row menu: Insert row below. Foot New row adds IN PLACE and reveals the row.
3. ROW PEEK: hover reveals an `Open` chip on the title cell (floating chip: border+
   shadow allowed; hit-area above cell inputs).
4. ENTITY SHEET (fixed right drawer, the chat-drawer language, ~440px): header =
   entity eyebrow + icon actions (open-as-panel ↗ · ⋯ duplicate/delete · ×) + serif
   editable title + PIPELINE PILLS from the first select field (click sets the cell
   LIVE; re-click clears) + underline facet tabs Fields · Page · Activity · n.
   Activity = per-row stream: notes (dot) and tasks (check + strike) + a Note/Task
   composer. Escape and click-away close the sheet, never the panel behind.

## VERIFY
- Active-tab menu shows all five actions; density persists per view after reload.
- CSV downloads with exactly the visible columns and the view's rows.
- Chip appears on hover, opens the sheet (no new panel, stage still); pill click
  updates the grid cell; a task toggles struck-through; `Open as panel` converts.
