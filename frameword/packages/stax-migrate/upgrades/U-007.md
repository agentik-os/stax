# U-007 — Foundations: foot family, fixed popovers, no native controls, no emoji chrome

TARGET: {{TARGET}}
REFERENCE: DESIGN-SPEC.md §5-§7 + frameword/apps/crm-specimen/src/styles.css

## DETECT
- Foot buttons of differing heights/colors; pin/unpin or metadata in a foot.
- Popovers clipped inside scroll containers (position: absolute under overflow).
- `<select>` for status-like choices; native date/time inputs; window.prompt renames.
- Emoji in UI chrome.

## APPLY
1. ONE foot-button family: ~30px, icon required, accent border; primary filled,
   destructive as red text. The foot is the ONLY action zone — no state toggles, no
   metadata lines.
2. Popovers inside scrollers are FIXED-positioned, anchored to the trigger rect,
   viewport-clamped, flip-up near the bottom; sticky cells get an explicit z-ladder.
   Escape closes the open popover, never the panel behind it.
3. Status-like choices = segmented buttons; dates/times = popover calendar + 30-min
   list; renames = borderless inline-edit; one accent-tinted focus border on inputs.
4. Chrome glyphs only: § ✶ → ⌄ ✓ ⋯ — no emoji.

## VERIFY
- getComputedStyle sweep of all foot buttons: one height, accent borders.
- Open every table/cell popover at the container's bottom edge: fully visible.
- Greps: zero `<select`, zero `type="date"`, zero `window.prompt`, zero emoji in chrome.
