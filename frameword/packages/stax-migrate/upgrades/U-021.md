# U-021: AI actions in the registry (computed insights first)

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (aiSay, tableInsight, kind "ai")

## DETECT
- AI features live as a detached chatbot: no panel declares what the agent can
  DO with its data; "AI" buttons are filled-accent CTAs fighting the primary.

## APPLY
1. Extend the action registry with kind "ai": the foot renders it as the ✶
   signature on an accent-SOFT chip (transparent border; never accent-filled —
   the L-FOOT gate counts fills). Palette and bridge list it automatically.
2. Each ai action's run() computes a REAL deterministic insight from the
   panel's own data first (table: rows/fields/filled%/select distribution;
   note: words + opening line; doc: title/subtitle/children recap) and hands
   it to the agent surface via one event (stax:ai) that opens the drawer and
   appends the insight ONCE. An app upgrades the same event to its LLM.

## VERIFY
- The ai button is accent-soft (computed bg != --accent) and the design gate
  still passes L-FOOT; clicking it opens the drawer with a message derived
  from the panel's actual data (change the data, the insight changes).
