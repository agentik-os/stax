# U-035: scanner v2 — the real world formats its code

TARGET: {{TARGET}}
REFERENCE: frameword/packages/stax-migrate/datascan.mjs (v2 core), tests/e2e-migration.test.mjs

## DETECT
- The data scanner greps LINE BY LINE: every Prettier-formatted chain
  (`supabase⏎.from("x")⏎.select()`), multi-line `useQuery(⏎ api.mod.fn`, and
  multi-line SQL policy silently VANISHES from the matrix — the worst failure
  mode, because it looks like a clean scan.
- Convex apps built on custom function builders (convex-helpers customQuery:
  authedQuery, adminMutation…) report zero functions.
- `fetch("/api/…")` call sites are invisible: a legacy endpoint the new front
  still calls is exactly what a migration forgets.
- Scan output is prose-only: CI and agents re-parse strings.

## APPLY
1. WHOLE-TEXT CORE: match over comment-stripped full source with an
   offset→line map (JS: `/*…*/` + `//`-to-EOL stripped, `http://` protected;
   SQL: `--` + block comments). `\s` then spans newlines, so real-world
   formatting is first-class and commented-out code never counts. Mask
   `storage.from(...)` spans BEFORE the table pass instead of a lookbehind.
2. CONVEX CUSTOM BUILDERS: any exported `= <ident>(` where ident contains
   query/mutation/action is a row (kind guessed from the name, evidenced as
   "custom builder"); `custom*` definitions are builders, not endpoints; every
   distinct wrapper is named in ONE warning for human confirmation.
3. FETCH→ROUTE LINKING: `fetch("/api/…")` sites bind to scanned routes with
   `[param]` segments matching anything; an unmatched fetch is a LEGACY-LEAK
   warning naming file:line. Projects with fetch sites and no route files get
   the warning too.
4. --json ON scan AND check: machine-readable output (layers/rows/warnings;
   pass/problems/drift) so CI and agents never scrape prose.
5. THE LOOP IS A TEST: tests/e2e-migration.test.mjs runs the REAL CLI on a
   temp fixture through init → scan --write → check exit 1 → map → check
   DATA 100% → drift → check exit 1, on every push, in milliseconds.

## VERIFY
- A fixture with a 5-line Prettier chain on a NEW table yields the table row
  with r/u ops; `// supabase.from("ghost")` and block-commented calls yield
  nothing.
- An `authedQuery` endpoint yields a query-kind row + the wrapper warning;
  the wrapper definition itself yields no endpoint row.
- A `fetch("/api/ghost")` with no matching route warns "legacy leak"; a
  matching fetch is counted on its route row's evidence.
- `data scan --json | jq .rows` parses; the e2e-migration test passes.
