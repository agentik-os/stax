# U-003 — Crumbbar utilities: bare theme icon right, repo link beside, topbar decluttered

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (crumbbar) · styles.css (.crumb-theme, .crumb-gh)

## WHY
Utility toggles are chrome, not content: they live at the crumbbar's RIGHT end as bare
24px icons (like the home glyph), not as bordered buttons in the topbar. The topbar
carries no duplicate of the org/brand already shown in the sidebar.

## DETECT
- Theme switcher rendered in the topbar, or styled as a bordered button.
- A brand/product name in the topbar duplicating the sidebar org switcher.
- External links (GitHub/docs) floating in the topbar.

## APPLY
1. Theme toggle → crumbbar right end: bare icon 24×24, no border, muted → foreground on
   hover; its menu opens UPWARD (bottom-anchored).
2. Repo/GitHub link → same bare-icon family, immediately LEFT of the theme icon.
3. Remove the topbar brand text if the sidebar already identifies the workspace.
4. Any control with no current effect renders DISABLED (dimmed, explanatory title),
   never hidden — availability is state, not existence; the bar never shifts.

## VERIFY
- Screenshot the crumbbar right end, both themes.
- getBoundingClientRect of the utility icons identical before/after opening a drill
  (no layout shift).
