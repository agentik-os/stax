# U-028: UX Wave 2 — thread memory, symmetric exits, identity, drag truth

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen @0.13.0 (resumeOrOpen, closing states)

## DETECT
Space switches reset to root (work discarded); overlays blink out; deep
panels wear anonymous type-only eyebrows; board drags are invisible mid-air
and silent on landing; the tour narrates without progress; the palette hides
its reach; personal spaces light the wrong dashboard chip.

## APPLY
1. PER-SPACE THREAD MEMORY: every settled path records as its space's resume
   point (session-scoped); every space entry point (sidebar, topbar menus,
   utility icons, palette) resumes the stashed thread when returning — with a
   "Resumed: <leaf>" toast. Opening the same space you are in stays a reset.
2. SYMMETRIC EXITS: palette, drawer and sheet leave in 140ms mirrors of their
   entries (fade/slide via a closing state that delays unmount).
3. EYEBROW IDENTITY: a panel without an authored eyebrow renders
   "<parent title> › <type>", never the bare type.
4. DRAG TRUTH: the dragged board card dims (.dragging), the target column
   rings in accent (inset 1px) over a stronger tint, and the drop flashes —
   a data write is never silent.
5. The tour narrates "TOUR i/n · … · ESC STOPS" and ends with a handoff;
   the palette's zero-query footer states its reach (spaces · rows · tasks ·
   notes · actions · ⌘↵ pinned); recents append their ancestor path; a
   dashboard nav chip only lights when the ACTIVE space belongs to it.

## VERIFY
- Deep thread → other space → back: the FULL thread returns (panel count
  proof) with the toast. Escape mid-exit: the sheet carries .out then
  unmounts. Deep eyebrows read "Parent › type". Nav chips are all dark in a
  personal space. The tour caption starts "TOUR 1/…".
