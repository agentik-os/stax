# U-036: the view deck, the field law, the nav conversion

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen (DtFootViews, dtb-phase, sb-label groups)

## DETECT
- The panel foot switches VIEW TYPES while named saved views hide in a toolbar
  tab row — two switchers, neither the Pipedrive/Moonbase pattern users know.
- Pipeline boards are FLAT: no phase bands over stage columns (SOURCING →
  ENGAGEMENT → DILIGENCE reads as one undifferentiated strip).
- A text field keeps the UA default (white bg, black ink) — invisible drift in
  light, a glaring white box in dark.
- Legacy menus get transplanted instead of rederived: grouped sections,
  sub-item navs and mega-menus land as ad-hoc chrome.

## APPLY
1. THE FOOT VIEW DECK: the foot's `.foot-seg` lists the collection's NAMED
   SAVED VIEWS as segments (Pipeline · Table · Tuesday…, each view owning its
   type/filters/density); clicking the ACTIVE segment opens the view-config
   pop UPWARD (rename · duplicate · View as · density · reset · delete;
   `top: "auto"` beats the pop class's top so bottom-anchoring works), `+`
   creates. The toolbar keeps ONLY search + filter chips + tools.
2. PHASE BANDS: a board view may declare `colGroups: [{label, options[]}]` —
   stage columns group under mono eyebrow + count bands with a 2px
   `--accent-2` underline; unnamed options trail unbanded. One accent family,
   never per-phase colors.
3. THE FIELD LAW (L-FIELD): every input/textarea/select is a tokenized
   surface — bg `--card`, hairline `--input` border, `--foreground` ink,
   `--muted-foreground` placeholder, accent-tinted focus border — in BOTH
   themes. The verify gate fails a light-bg field in dark mechanically; the
   e2e suite carries the canvas-luminance case.
4. NAV CONVERSION: rederive the legacy menu by the spec's table — product
   areas → dashboards, grouped sidebar sections → SpaceGroups (one mono
   `.sb-label` per group), an item with sub-items → a space whose children
   ARE the sub-items, mega-menu → the palette, page tabs → sibling drills,
   breadcrumbs → the crumbbar, user menu → avatar menu, badges → sidebar
   KPIs. M9's phase 1 opens with this NAV MAP.

## VERIFY
- The data panel foot reads the named views (e.g. All customers · By plan ·
  Pipeline · +); switching to a colGroups board renders phase bands with
  correct per-band counts; the active segment's second click opens the pop
  ABOVE the foot and every slot is clickable.
- In dark, the playground/chat/invite composer field measures luminance < 0.5
  (canvas-converted); injecting `background: white` fails the e2e case.
- The sidebar space list shows one mono section label per SpaceGroup.
