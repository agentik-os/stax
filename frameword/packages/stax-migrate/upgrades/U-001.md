# U-001 — Engine: pinned roots · reload keeps pins · Back/Forward rewind

TARGET: {{TARGET}}
REFERENCE: github.com/agentik-os/stax — packages/panels-core + packages/panels-react (≥ 0.1.0)

## WHY
Three engine-level guarantees shipped after the initial grammar: a pinned Space ROOT
detaches into the reference rail instead of dying when its space is replaced; a page
reload can no longer destroy the rail; and the browser Back/Forward buttons rewind the
panel thread instead of exiting the app.

## DETECT (run each; a hit = the upgrade applies)
- `grep -rn "reconcileLocation(emptyWorkspace()" <bindings>` — reload rebuilds from
  empty: the rail is destroyed on refresh.
- `grep -rn "history.replaceState" <bindings>` with NO `popstate` listener — Back exits.
- pinPanel guards `role === "root"` — roots cannot be pinned.

## APPLY
1. Update `@frameword/panels-core` and `@frameword/panels-react` to ≥ 0.1.0 (or re-vendor
   their `src/` from the reference repo if you vendored).
2. In the provider: restore the device snapshot FIRST, then reconcile the URL ON TOP of
   it — `reconcileLocation(storedSnapshot, loc)` (openPath reveals-never-duplicates).
3. Opt the app into history: `urlSync="push"` on WorkspaceProvider.
4. Surface the root pin: render the PIN button on root panels too (pin state for a root
   is `panel.pinned`, not retention), with the P shortcut allowed on roots.

## VERIFY (evidence required)
- Pin a detail AND a root, switch dashboard, reload → `referenceRailOrder.length`
  unchanged, both refs render. Paste the state dump.
- Drill 2 deep, press browser Back twice → thread rewinds in-app, never leaves the site.
- Laws tests green if you consume the packages' test suite.
