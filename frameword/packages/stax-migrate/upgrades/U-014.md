# U-014: menu design v2 · XXL exact-width · double-click align

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/styles.css (.tp-slot, .dp-pop, .tb-sep)
+ App.tsx (isFluid, dblclick bar, scroll-on-resize)

## DETECT
- Menu items in mono, or selected items as filled accent slabs.
- XXL (or any max size) computed as "remaining space" — wide content pushes past the
  viewport, or gutters are asymmetric when aligned.
- No gesture to align a panel with the view; changing a panel's size leaves it
  half-off-screen. Sizes ignored on pinned references.

## APPLY
1. MENUS: items sans 500 (body-1px), 6/9 padding radius 6; group labels mono
   uppercase; selected = accent-soft bg + accent text + right ✓ (segments: soft fill,
   no check, centered); container padding 5 radius 10. Purge legacy hover rules that
   fight the new ones (grep duplicated selectors).
2. XXL: `calc(100% - stage-pad)` of the scroll container (exact visible width,
   symmetric gutters); min-width 0 so inner content scrolls internally; a size change
   re-scrolls the stage to the leaf; XXL applies to pinned references too (fluid).
3. Double-click on any panel bar (buttons exempt) aligns that panel at the stage pad
   and focuses it. Scroll alignment uses the stage's REAL pad, never hardcoded margins.

## VERIFY
- Screenshot a menu: sans items, mono labels, soft ✓ selection.
- XXL panel: left gutter == right gutter (rect proof); table scrolls inside.
- Dblclick a mid-thread bar: panel.left - stage.left == pad (±1px).
