# U-011: panel setups remembered · root-respecting links · drill polish · no-dash copy

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen/src/App.tsx (sizePrefs, mainClosed/deepLink, drill CSS)

## DETECT
- A user-chosen panel width is lost when the panel closes or the page reloads.
- With the space's MAIN panel deliberately closed (promoted child leading), a sidebar
  or palette link resurrects it.
- Drill-row hover paints a full-bleed squared slab; numbers sit in boxed tiles.
- UI copy peppered with " — " appositions.

## APPLY
1. SIZE MEMORY: a tiny device-local store keyed by resourceKey ("frameword-sizes"
   pattern); the panel's width override reads/writes it, so every panel reopens in its
   last chosen size. Presentation only, never navigation state.
2. ROOT RESPECT: deep-links into the CURRENT space drop the space root from the chain
   when the main was closed (the promoted chain leads); ONLY the sidebar head row
   reopens the main, explicitly, rebuilding [root, ...current chain].
3. DRILL POLISH: flat mono index (no tile), hover = INSET rounded fill (::before,
   inset 4px -10px, radius 10) with the index and arrow turning accent and the arrow
   sliding 3px; 130ms ease.
4. COPY: purge " — " from every UI string; prefer ":", "·" or a period.

## VERIFY
- Set a panel to XL, close it, reopen, reload: still XL (three measurements).
- Close the main, click two sidebar children: the main never reappears; head click
  brings it back above the chain.
- Hover screenshot; `grep -rn " — " src` → zero in UI strings.
