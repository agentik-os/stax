# U-010 — Root-close promotion: the page never vanishes under you

TARGET: {{TARGET}}
REFERENCE: packages/panels-core closePanel (root branch) + laws tests

## WHY
Closing the ROOT used to end the whole space — with drills open, users landed on an
empty stage. New semantics: closing the root PROMOTES its attached child to root (same
space, same thread, refs untouched); the space ends only when the root is the last
context panel. A pinned root still detaches to the rail while its child takes the lead.

## DETECT
- In the engine: closePanel on role="root" nulls spaceId even when attached children
  exist.

## APPLY
1. Engine: in closePanel's root branch, find the attached child; if present — old root
   detachOrDelete (pinned → rail), heir.parentInstanceId = null, role = "root",
   retention = "retained", pinned = false; rootInstanceId = heir; fix leaf/focus if they
   pointed at the old root. No branch policy on the kept subtree. If NO child — previous
   end-the-space behavior.
2. UI: the root's × tooltip becomes "Close panel — the next panel takes the lead" when
   the thread is deeper than 1.
3. Update any doc copy promising "the root's × closes the whole Space".

## VERIFY
- Laws tests: chain of 3, close root twice → ["b","c"] then ["c"], spaceId unchanged;
  third close ends the space. Pinned root close → old root in the rail + child leads.
- Live: open a drill, close the main panel → you stay on the page.
