# U-029: UX Wave 3 — the sheet's full grammar, hairline slabs, power gestures

TARGET: {{TARGET}}
REFERENCE: frameword/apps/crm-specimen @0.14.0

## DETECT
The record sheet wears underline tabs and has no foot; table headers, calc
bands and board/kanban columns are gray slabs; dates render as filled pills;
no range select, no digit navigation; crumb clicks destroy the thread; the
tour hides in the palette on first run.

## APPLY
1. SHEET GRAMMAR: facets become FOOT segments in a real sheet foot (44px,
   border-top, secondary bg); the foot riffles siblings (‹ › + ArrowUp/Down
   outside inputs, mono i/n position). The underline-tab motif dies.
2. HAIRLINE SLABS: thead/tfoot sit on the CARD surface over one hairline
   (sticky keeps the card bg as its mask); board and kanban columns are
   transparent with hairline gutters (+ .over feedback stays transient).
3. CHIP GRAMMAR: a chosen date is bare mono text (hover = secondary fill),
   never the app's only filled pill.
4. POWER GESTURES: digits 1-9 click the focused panel's Nth drill row (the
   painted index IS the key); shift-click selects the row RANGE in table
   order; crumb click = focus + scroll into view, ⌥-click = rewind (title
   documents both).
5. FIRST RUN: the Overview foot offers "Play the tour" (quiet secondary)
   until the tour has run once (localStorage flag, set by any tour start).
6. The drawer's injected AI insight shows the typing beat (450ms) before it
   lands — same loading grammar as everything else.

## VERIFY
- Sheet: segments in the foot, no .sh-tabs, ‹ › and arrows move i/n.
- thead bg == panel bg; board columns transparent with 1px gutters.
- Date values compute mono/transparent. Digit 2 opens the second drill.
- Shift-click row 1 then row 4: "4 selected". Crumb click keeps the panel
  count; ⌥-click rewinds. Fresh profile: the CTA shows, plays, disappears.
